1. What is OOP?

OOP stands for Object-Oriented Programming.

It is a programming style where we use objects and classes to organize code.

Instead of writing everything in one place, we divide the program into small parts.

Example:

Think of a car 
Car is an object.
It has:

* Properties → color, speed
* Functions → start(), stop()

Same concept is used in programming.

2. Classes and Objects

Class = blueprint (design)
Object = real thing created from class

Example Code:

#include <iostream>
using namespace std;

class Student {
public:
string name;
int age;
};

int main() {
Student s1;

```
s1.name = "Naeem";
s1.age = 20;

cout << s1.name << endl;
cout << s1.age << endl;

return 0;
```

}

Explanation:

class Student → defines structure
s1 → object of class
s1.name → accessing data

---

3. Access Specifiers

---

There are 3 types:

1. public → accessible everywhere
2. private → accessible only inside class
3. protected → used in inheritance

Example:

class Test {
public:
int a;

private:
int b;
};

Explanation:

a can be used anywhere
b can only be used inside class

---

4. Constructors

---

Constructor is a special function that runs automatically when object is created.

Example Code:

#include <iostream>
using namespace std;

class Student {
public:
string name;

```
Student() {
    cout << "Constructor called" << endl;
}
```

};

int main() {
Student s1;
return 0;
}

Explanation:

Constructor name = class name
No return type
Runs automatically

---

5. Parameterized Constructor

---

Constructor that takes values.

Example Code:

#include <iostream>
using namespace std;

class Student {
public:
string name;
int age;

```
Student(string n, int a) {
    name = n;
    age = a;
}
```

};

int main() {
Student s1("Naeem", 20);

```
cout << s1.name << endl;
cout << s1.age << endl;

return 0;
```

}

6. Encapsulation

---

Encapsulation means hiding data and controlling access.

We use private variables and public functions.

Example Code:

#include <iostream>
using namespace std;

class Bank {
private:
int balance;

public:
void setBalance(int b) {
balance = b;
}

```
int getBalance() {
    return balance;
}
```

};

int main() {
Bank b1;

```
b1.setBalance(5000);
cout << b1.getBalance();

return 0;
```

}

Explanation:

balance is hidden
access through functions

---

7. Inheritance

---

Inheritance means one class can use properties of another class.

Example Code:

#include <iostream>
using namespace std;

class Animal {
public:
void speak() {
cout << "Animal speaks" << endl;
}
};

class Dog : public Animal {
};

int main() {
Dog d;
d.speak();

```
return 0;
```

}

Explanation:

Dog class uses Animal class
Code reuse happens

---

8. Polymorphism

---

Polymorphism means one function works in different ways.

Example (Function Overloading):

#include <iostream>
using namespace std;

class Math {
public:
int add(int a, int b) {
return a + b;
}

```
int add(int a, int b, int c) {
    return a + b + c;
}
```

};

int main() {
Math m;

```
cout << m.add(2,3) << endl;
cout << m.add(2,3,4);

return 0;
```

}

Explanation:

Same function name
Different parameters

---

9. Abstraction

---

Abstraction means showing only important details and hiding complexity.

Example:

When you use ATM:
You press buttons
But you don’t know internal system

Same in programming.

Example Code:

class Shape {
public:
virtual void draw() = 0;
};

Explanation:

User only sees draw()
internal logic hidden

---

10. This Pointer

---

this pointer refers to current object.

Example Code:

#include <iostream>
using namespace std;

class Test {
public:
int x;

```
void setX(int x) {
    this->x = x;
}
```

};

int main() {
Test t;
t.setX(10);

```
cout << t.x;

return 0;
```

}

Explanation:

this->x means current object variable
